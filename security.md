### Security Risks
    - **Users**: Intended users of the code in this repository include myself (developer), and students within CSE-3000.
    - **Risks**: If this code/data got into the hands of someone malicious, in a simple case a student taking the class next semester who wants to see the answers to all assignments, they could easily view this repo. They could also attempt to make pull requests into this repository in order to annoy me. 
    - **Precautions**: In terms of people viewing code for answers, my solutions have to be within this repo in order to be graded, and to my knowledge making the repo private would cause issues with this. For this reason & with brief cost benefit analysis, I don't think it's necessary to make this repo private or about code viewing. All assignments can be promptly gen-AI'd, and they'd have no clue what I got as a grade on these assignments anyway, so there's no real incentive to copying from me. In terms of making pull requests, I have branch protection enabled so no user-submitted changes can affect my workflow.
